This is an **Update to the Phase 2A Refactor Execution Map** based on the analysis of the provided MATLAB code (`ssASM3_DR_datagen.m`, `benchmarkinit_ASM3_DR.m`, and `Data_writer_reac_energy.m`).

### 1. MATLAB Interface Analysis

The analysis reveals that the existing `ssASM3_DR_datagen.m` is unsuitable for the new architecture because it contains a hardcoded infinite loop (`while true`), clears the workspace aggressively (`clear all`), and manages its own state via `.mat` files.

**Key Findings:**

* **Initialization:** `benchmarkinit_ASM3_DR.m` loads hardcoded files (`KLa3_Setpoints_nominal.mat`). The Python runner must override these variables *after* initialization but *before* simulation start.
* **Energy Calculation:** `Data_writer_reac_energy.m` calculates energy internally (columns 15 in the output CSVs). The Python pipeline **should not re-calculate energy physics** (kLa * Volume * Power) but should instead aggregate the `total_energy` column provided by MATLAB.
* **Output Pathing:** `Data_writer_reac_energy.m` has a hardcoded absolute path (`/Users/ikai/...`). This **must be patched or overridden** dynamically to support the new `data/raw/{run_id}` directory structure.
* **Simulation Mode:** The plan requires *dynamic* step-response data, but `ssASM3_DR_datagen.m` focuses on *steady-state* reliability points. We need a new MATLAB driver function.

### 4. Updated Pipeline Diagram

**1. Scenario Gen (Python):** Defines `Scenario_ID`, `KLa_Reduction_%`, `Duration`.
**2. Setup (Python):**

* Generates `KLa_Setpoints_{ID}.mat` (Time-series vectors).
* Creates directory `data/raw/{ID}/`.
**3. Execution (MatlabAdapter):**
* Calls `run_dynamic_step('KLa_Setpoints_{ID}.mat', 'data/raw/{ID}/')`.
**4. Simulation (MATLAB):**
* `benchmarkinit_ASM3_DR` (Loads defaults).
* `load('KLa_Setpoints_{ID}.mat')` (Overrides defaults).
* `sim(...)`.
* `Data_writer_reac_energy(...)` (Writes `reactorX_data.csv` to `data/raw/{ID}/`).
**5. ETL (Python):**
* Reads `reactorX_data.csv`.
* Extracts `Col 15` (Energy) and `Col 10` (S_NH4).
* Saves to `processed/simulation_database.parquet`.
